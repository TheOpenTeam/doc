## 存储类型
| Java 类型 | Rust 类型 | Minecraft 存储方式 |
|-----------|-----------|-------------------|
| `boolean` | `bool` | 1 字节，`0x01` 为 true，`0x00` 为 false |
| `byte` | `i8` | 1 字节，有符号 |
| `short` | `i16` | 2 字节，大端序 |
| `int` | `i32` | 4 字节，大端序 |
| `long` | `i64` | 8 字节，大端序 |
| `float` | `f32` | 4 字节，大端序 IEEE 754 |
| `double` | `f64` | 8 字节，大端序 IEEE 754 |
| `String` | `String` | VarInt 长度前缀 + UTF-8 字节 |
| `VarInt` | `i32` | 1-5 字节，变长编码 |
| `VarLong` | `i64` | 1-10 字节，变长编码 |
| `UUID` | `[u8; 16]` 或 `uuid::Uuid` | 16 字节，大端序 |
| `Position` | 自定义结构体 | 64 位打包：x(26位) + z(26位) + y(12位) |
| `Angle` | `u8` | 1 字节，旋转角 (360/256 度) |
| `Byte Array` | `Vec<u8>` | VarInt 长度前缀 + 字节数据 |
| `NBT Tag` | `nbt::Value` | 二进制 NBT 格式 |
| `Optional<T>` | `Option<T>` | 布尔标志 + 存在时的 T |
| `Slot` | 自定义结构体 | 物品数据，格式较复杂 |
| `Entity Metadata` | 自定义结构体 | 特定格式的元数据列表 |

### VarInt/VarLong的编码、解码
`varint`与`varlong`都在主crate的`utils`模块下。  
```rust
use mcproto::utils::varint;
let mut buf = Vec::new();
varint::encode(114514, buf);
```
`encode`函数需要传入一个`i32`整数类型和一个实现`Write` trait的任意buf。它会将结果写入该buf中。  
注意，在VarInt中，**数据最多存储5个字节**，如果所传入的数值超出了i32的范围便会得到`CodecError:VarTooLarge`错误。  
而在VarLong中，**数据最多存储10个字节**，如果所传入数值超出i64范围，与上文同理。  
而`decode`函数需要传入一个实现`Read` trait的buf。
无论`encode`还是`decode`，都会返回一个`Result<(), CodecError>。  
```rust
varint::decode(&mut buf.as_slice()) // 得到114514
```
