> 为了方便开发者和用户测试代码是否可以正常运行，和自身电脑的性能如何，我们提供了**不同的测试文件**。

### 已有的测试列表
| 测试名(函数名) | 位置 | 作用 |
|-----------|-----------|-------------------|
| `test_encode` | `mcproto/tests` | 测试VarInt编码（1亿次） |
| `test_both` | `mcproto/tests` | 测试VarInt编码+解码（1亿次） |
| `test_encode_varlong` | `mcproto/tests` | 测试VarLong编码（1亿次） |
| `test_both_varlong` | `mcproto/tests` | 测试VarLong编码+解码（1亿次） |
| `serverbound_packet_test` | `mcproto/derive_tests` | 测试`ServerboundPacket` derive和自动生成的`encode`、`decode`方法是否可用|

### 如何使用测试
如果想跑**所有Crate的所有测试**，使用：
```bash
cargo test --workspace --release -- --nocapture
```

?> -- --nocapture可以使`println!`宏输出。

?> --release可以使用`内联`功能，性能提高很大。

每一项测试都有对应的输出结果。若发现结果异常或无法测试成功，请及时提出`Issues`.
